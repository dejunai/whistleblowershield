## Suggested Improvements
Refine these to boost LLM compliance, reduce edge-case errors, and streamline ingest:

- **RUN SCOPE Validation**: Strengthen the abort logic. Add: "If Records Requested > 20, process only the top 20 by relevance and note the truncation in `json_run_notes`. Free-tier models can't handle massive batches reliably." This prevents overload-induced hallucinations.
  
- **Source Confidence Threshold**: In the URL/CITATION sections, add a quick decision tree:
  ```
  Confidence levels:
  - High: Official .gov or verified legalsource (e.g., Justia, Casetext).
  - Medium: Reputable secondary (e.g., Nolo, law firm summary)—include but flag in _review_notes.
  - Low: Omit entirely.
  Always prefer official; if none exists after 3 targeted searches, omit.
  ```
  This gives the LLM a mental checklist without rigidity.

- **Taxonomy Proposals**: Add a cap: "Limit `new_terms_proposed` to 5 per batch. Prioritize by frequency across records." Prevents spam. Also, clarify: "If a close-but-not-perfect existing slug exists (e.g., 80% fit), use it and note the approximation in `json_run_notes` rather than proposing."

- **Integrity Block**: Make `with_errors: true` mandatory if `record_count < records_requested`. Expand error examples:
  ```
  - "CA-1102.5: limit_value unconfirmed across sources; left ambiguous."
  - "No primary_agency for PR-115; local variations suspected."
  ```
  Add: "If with_errors: true, set meta.batch_completed to include 'INCOMPLETE' suffix, e.g., '2026-03-20 18:00 UTC (INCOMPLETE)'."

- **Model-Specific Tweaks**: In `meta.source_name`, suggest: "Include context window size if known (e.g., 'Grok-4.1 free tier, 8k tokens)'." Helps humans gauge completeness.

- **JSON Schema Nitpicks**:

## Why Constraining Fails
Tight prompts work for narrow extraction (e.g., "parse this exact sentence") but collapse under open research like yours. LLMs aren't databases—they're pattern-matchers predicting "complete" outputs. When handcuffed, they prioritize fitting the mold over truth, hallucinating to avoid "incomplete" outputs that feel like failure.

## Reward-Based Wins
Your forgiveness-first pivot flips this:
- **Expected Gaps**: "Empty is correct" reprograms the model to see omission as success.
- **Human Backup**: "First pass only" lowers stakes, letting the model relax.
- **Self-Audit**: `integrity` block turns errors into assets for reviewers.
