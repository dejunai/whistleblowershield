## üî¥ Launch Blockers

Issues that will cause silent failures, broken queries, or data inconsistencies in production.

### Data / Query Integrity
1. **`register-taxonomies.php`** ‚Äî `jx-citation` missing from `ws_disclosure_type` object_types. Citations write term relationships via `save_terms => 1` but the taxonomy isn't registered against the CPT ‚Äî `tax_query` on citations for disclosure type will return nothing.
2. **`acf-jx-statutes.php`** ‚Äî All taxonomy fields use `save_terms => 0` while citations and interpretations use `save_terms => 1`. A `tax_query` across statutes and citations for the same taxonomy (e.g. `ws_disclosure_type`) is impossible ‚Äî statutes require `meta_query`, citations require `tax_query`. Almost certainly unintentional. Requires a meta migration pass on existing statute posts after fix.
3. **`acf-jx-interpretations.php`** ‚Äî `ws_jx_interp_affected_jx` has `load_terms => 0`. The auto-population hook writes to post meta directly, so editors who try to manually override affected jurisdictions will see a blank field even when meta is populated.

### ACF Field Key Collisions
4. **`acf-stamp-fields.php` / `acf-jurisdictions.php`** ‚Äî `field_create_author`, `field_date_created`, `field_last_edited`, `field_last_edited_author` registered as duplicate ACF field keys across both files. ACF may log warnings or suppress one registration silently.
5. **`acf-plain-english-fields.php` / `acf-jx-summaries.php`** ‚Äî `field_plain_english_reviewed`, `field_plain_english_by`, `field_plain_english_reviewed_by` all have duplicate keys across both files. Same collision risk.

### Missing/Broken Wiring
6. **`ws-core.php`** ‚Äî `ws_feed_monitor_deactivate` registered as a deactivation hook with no confirmed corresponding function. Will fatal on deactivation if the function doesn't exist.
7. **`loader.php`** ‚Äî LOADING STRATEGY docblock lists ACF as step 2 and query layer as step 3, but the actual load order is CPT ‚Üí Query ‚Üí Taxonomies ‚Üí ACF. Misleads anyone tracing dependency failures.

## üü° Fix Before Launch

Real issues that won't crash the site but will cause editorial confusion, query inconsistencies, or silent data drift.

### Editorial / UI
8. **`acf-jx-statutes.php`** ‚Äî `field_jx_statute_last_reviewed` is declared outside all tabs. It will render loose at the bottom of the metabox, invisible to editors working inside the tab structure.
9. **`acf-source-verify.php`** ‚Äî `ws-legal-update` absent from location rules with no documented exclusion rationale. If legal updates are ever ingested programmatically, there's no source provenance tracking on them.
10. **`acf-jx-summaries.php`** ‚Äî Version history has no `3.6.0` entry for the `ws_auto_` rename of `plain_english_by` and `plain_english_reviewed_by`. Unclear whether those keys were renamed or silently inconsistent.

### CSS / Frontend
11. **`ws-core-front-jx.css`** ‚Äî `[ws_jx_interpretation]` added in v3.8.0 is absent from the file header and has no stub ruleset. The interpretation section renders with zero scoped styles.

### Documentation / Convention Drift
12. **`acf-stamp-fields.php`** ‚Äî FIELDS section of the docblock still shows pre-3.6.0 unprefixed meta key names (`date_created`, `last_edited`, etc.) instead of the current `ws_auto_*` names.
13. **`ws-core-front.js`** ‚Äî Version history tops out at `2.4.0` inside a `3.8.0` plugin. No `3.8.0` entry noting the CSS split.
14. **`register-glossary.php`** ‚Äî TRANSIENT docblock omits `delete_ws_glossary` from the cache invalidation hook list. A deleted term remains in cache until TTL expiry ‚Äî phantom tooltips.
15. **`register-glossary.php`** ‚Äî ACF group key `group_ws_glossary_term` violates v3.1.0 naming rules (should be `group_glossary_metadata`, no `ws_` prefix, ends with `_metadata`).
16. **`ws-core.php`** ‚Äî `WS_JURISDICTION_TERM_ID` constant naming acknowledged as confusing inline but has no `@todo pre-launch` tag. Won't surface in a pre-launch audit sweep.
17. **`ws-core.php`** ‚Äî `is_singular() || is_page()` in `ws_core_enqueue_assets()` ‚Äî `is_page()` is a subset of `is_singular()`, dead code.
18. **`loader.php`** ‚Äî Assembly layer `error_log`-only handler (no `admin_notices`) has no comment explaining why. Someone will add one, wonder why it never fires.
19. **`loader.php`** ‚Äî ACF admin-only load has undocumented REST API / WP-CLI blind spot. Needs `@todo pre-launch` before Phase 2 REST work begins.
20. **`register-taxonomies.php`** ‚Äî `ws_seed_disclosure_taxonomy()` doesn't use `ws_bulk_insert_hierarchical()` ‚Äî undocumented. Risk of silent breakage if someone "refactors" it to match the helper's expected data shape.
