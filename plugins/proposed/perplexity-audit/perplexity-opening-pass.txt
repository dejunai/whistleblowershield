## Overall Structure Assessment

The architecture is well-designed for its scale — layered loading, query-layer discipline, and extensive docblocking are all good practice . That said, several concrete bugs and oversights surfaced across the two core files.

***

## Bugs & Oversights Found

### 🔴 Critical / Functional Bugs

**1. `WS_JURISDICTION_TERM_ID` constant naming is actively misleading**
The constant is named `WS_JURISDICTION_TERM_ID` but stores the string `'ws_jurisdiction'` (a taxonomy *slug*), not an integer term ID . The comment admits this directly. The risk: any future developer who writes `get_term( WS_JURISDICTION_TERM_ID )` expecting an integer will silently get wrong behavior. The constant should be renamed `WS_JURISDICTION_TAXONOMY` or `WS_JURISDICTION_TAX_SLUG` before launch to eliminate the cognitive trap.

**2. `is_singular()` + `is_page()` redundancy in asset enqueue**
In `ws_core_enqueue_assets()`, the condition is `if ( is_singular() || is_page() )` . `is_page()` is entirely redundant — `is_singular()` already returns `true` for pages (`is_singular()` covers posts, pages, and all CPTs). This is harmless now but is a latent bug magnet: it implies `is_page()` is doing something `is_singular()` misses, which could lead someone to incorrectly extend the pattern (e.g. adding `is_archive()` thinking it's also needed).

**3. Assembly layer shortcode errors are silent to the frontend**
In `loader.php`, the render and shortcode file error blocks do `error_log()` only — no `admin_notices` call is appended for missing render/shortcode files  (unlike CPT, Query, Taxonomy, Matrix, ACF, and Admin blocks, which all fire `admin_notices`). A missing `render-section.php` would silently produce a blank jurisdiction page with no admin warning. This is an oversight in the v3.6.0 split.

***

### 🟡 Design Oversights / Tech Debt

**4. Loader loading order lists taxonomies *after* CPT and Query layers, but the comment says order is non-negotiable**
The `loader.php` comment block says "LOAD ORDER IS NON-NEGOTIABLE: taxonomies must be registered before any seeder runs" — but this only applies to the Admin Layer . In the Universal Layer, CPTs load first, then Query, then Taxonomies. If any CPT registration file references a taxonomy slug (e.g. in its `taxonomies` array), that slug won't be registered yet when the CPT registers. WordPress handles this gracefully at runtime due to `init` hook timing, but the load order comment creates false confidence that the order in the Universal Layer is safe for all cases.

**5. `ws-core-front.js` filter tab ARIA state is incomplete**
The filter button click handler updates `.ws-active` CSS class but never updates `aria-pressed` or `aria-selected` on the buttons . Keyboard and screen reader users will correctly reach the buttons (they're `<button>` elements presumably), but the active/inactive state won't be announced by assistive technology. This is a real accessibility gap in a plugin whose content is legally sensitive.

**6. No nonce verification on any deactivation hook**
`ws_url_monitor_deactivate` and `ws_feed_monitor_deactivate` are registered via `register_deactivation_hook()` . WordPress handles CSRF protection on these via `check_admin_referer()` internally — but if either deactivation function performs any DB writes (which URL and feed monitors typically do to clean up scheduled events or option keys), those should be verified inside the functions themselves. This is worth auditing inside those files.

**7. `WS_CACHE_LEGAL_UPDATES_SITEWIDE` transient has no TTL documentation**
Three cache constants are defined but only two (`WS_CACHE_ALL_JURISDICTIONS`, `WS_CACHE_JX_INDEX`) have their invalidation strategy documented inline . `WS_CACHE_LEGAL_UPDATES_SITEWIDE` is documented as "invalidated on every ws-legal-update save" but there's no note on what TTL is set at creation time, or what happens if a save fails mid-flight leaving a stale transient.

***

### 🔵 Minor / Style Issues

**8. Loader tab/space inconsistency**
The Universal Layer uses tabs for indentation inside `foreach` blocks; the Admin Layer switches to 4-space indentation for the `$acf_files` loop . Not a bug, but it will flag in any PHP_CodeSniffer run.

**9. `matrix-jurisdictions` is loaded but `matrix-agencies` is also in the matrix list without a documented dependency note**
The loader comment says `matrix-jurisdictions.php` must run before agency/fed-statutes/assist-org matrices because the US term must exist first . But `matrix-agencies.php` is listed *after* `matrix-assist-orgs.php` in the array. If `matrix-agencies` depends on the US term (same as the others), its position is fine — but if there's a dependency *between* `matrix-agencies` and `matrix-assist-orgs`, it's undocumented and the order would be wrong.
