### 🟡 Design Oversights / Gaps

**4. `matrix-agencies.php` — `WS_MATRIX_SEEDING_IN_PROGRESS` constant defined *inside* the foreach loop**
In `ws_seed_agency_matrix()`, the `define( 'WS_MATRIX_SEEDING_IN_PROGRESS', true )` guard is inside the `foreach` loop body . In `ws_seed_assist_org_matrix()` it is correctly placed *before* the loop . In `ws_seed_fed_statutes_matrix()` it is also inside the loop body . Defining a constant inside a loop after the first iteration has no effect (constants can't be redefined), but it means on the very first post creation the constant is not yet set when `wp_insert_post` fires — which is the whole point of the guard. The constant needs to be defined before `wp_insert_post` is called, not inside the loop that calls it. Move all three `define()` calls to before the `foreach`.

**5. `matrix-agencies.php` — no `ws_disclosure_type` or `ws_process_type` taxonomy assignment**
The `ws-agency` CPT is registered against `ws_disclosure_type` and `ws_process_type` in `register-taxonomies.php` . The assist-orgs matrix assigns `ws_disclosure_type` terms per record . But `ws_seed_agency_matrix()` writes no taxonomy assignments beyond `ws_jurisdiction` and `ws_languages`  — no disclosure types, no process types. For agencies like SEC (securities fraud) or EPA (environmental), this means the taxonomy filter panel will never surface them via disclosure category. Either the agency data needs a `disclosure_types` key per entry, or the design decision to exclude them needs to be documented.

**6. `matrix-assist-orgs.php` — `ws_aorg_services` written as a PHP array into post meta directly**
The seeder writes `'ws_aorg_services' => $org['services'] ?? []` as a raw PHP array via `update_post_meta()` . ACF serializes arrays correctly on its own saves, so this will produce a PHP-serialized string in the meta table that ACF can read back. However, the serialized format from `update_post_meta()` on an indexed array won't include ACF's `field_` key reference — meaning the field value will load correctly in the ACF UI only if ACF doesn't rely on the hidden `_ws_aorg_services` shadow key to resolve the field. If ACF uses the shadow key to detect "this meta belongs to a field", it won't find it for seeder-written meta and may overwrite with an empty value on first admin save. All other array fields in this seeder (`services`) share this risk. The fix is to also write `update_post_meta( $post_id, '_ws_aorg_services', 'field_ws_aorg_services' )` (or whatever the ACF field key is) alongside every array meta write.

**7. `matrix-state-courts.php` — `'other'` sentinel comment says "both matrices" but `matrix-federal-courts.php` may have a diverged definition**
The comment on the `'other'` entry says "Defined in both matrices so 'other' is available regardless" and "the state matrix entry overwrites the federal one — both are identical" . This is only safe if both definitions truly stay in sync — there's no code enforcing it. If `matrix-federal-courts.php`'s `'other'` entry is ever updated (e.g. `level` changed), the state matrix's `'other'` silently wins in the merged select due to array key collision. A `@see` cross-reference comment in both files, or extracting `'other'` to `matrix-helpers.php` as a shared constant, would eliminate the risk.
