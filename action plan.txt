Phase 0 — Global Architectural Rules

These rules apply to all phases.

Query Layer Boundary (Non-negotiable)

Shortcodes are presentation only.

They must never call:

get_field()
get_post_meta()
WP_Query

All data must come through:

query-jurisdiction.php

If a shortcode needs data not currently returned:

Extend the relevant ws_get_*() function

Then update the shortcode

Never bypass the query layer.

Matrix Seed Contract

All matrix-seeded records must include:

ws_matrix_source

Editorial edits must trigger divergence tracking.

Unified Option-Gate Method

All matrix seeders and taxonomy seeders must use this exact pattern. No other gating format is permitted.
Key format: ws_seeded_{seeder_slug}

if ( get_option( 'ws_seeded_{seeder_slug}' ) !== '1.0.0' ) {
    ws_{seeder_slug}_seed();
    update_option( 'ws_seeded_{seeder_slug}', '1.0.0' );
}

Note: there is no live data to protect.

Phase 3 — Jurisdiction Architecture Refactor
This is the largest structural change. It replaces the ws_jx_code meta join with the ws_jurisdiction taxonomy, updates the attach flag pattern on jx-citation to use the new taxonomy, and propagates the complete attach flag pattern to jx-statute and jx-interpretation.

Phase 3.1 — Register ws_jurisdiction Taxonomy
In register-taxonomies.php, register the new taxonomy:
Properties:

Slug: ws_jurisdiction
Private, non-hierarchical
Object types: jx-statute, jx-summary, jx-citation, jx-interpretation, ws-agency, ws-assist-org

Seed terms from jurisdiction-matrix.php using the unified option-gate method:

Key: ws_seeded_jurisdiction_taxonomy
Initial value: 1.0.0


Phase 3.2 — Retire ws_jx_code
Remove ws_jx_code as the jurisdiction join mechanism across the plugin.
Files to update:

acf-jurisdiction.php — remove or archive the ws_jx_code field definition
query-jurisdiction.php — replace all ws_jx_code meta queries with ws_jurisdiction taxonomy queries
admin-relationships.php — remove any sync logic tied to ws_jx_code
admin-hooks.php — remove any hooks writing or reading ws_jx_code

Add ws_jx_code to the Phase 11.1 artifact scan list.

Phase 3.3 — Update jx-citation Attach Flag Pattern
jx-citation already has an attach_flag implementation but it currently references ws_jx_code. Update it to use ws_jurisdiction taxonomy.
ACF field updates in acf-jx-citations.php:

attach_flag — toggle, no change to field key
order — integer, conditionally exposed when attach_flag is on, no change to field key
Remove any field referencing ws_jx_code

Query layer update in query-jurisdiction.php:
ws_get_jx_citation_data() must:

Accept a jurisdiction taxonomy term as its scope parameter, replacing ws_jx_code
Filter by attach_flag set to true
Filter by is_published — only return published posts
Sort results by order ascending
Return all plain language fields (per Phase 9 spec)

Render integration:

Confirm ws_render_section() calls ws_get_jx_citation_data() with the taxonomy term
Citation list renders independently, sorted by order
