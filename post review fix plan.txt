Post-Review Remediation Plan
Phase 1 — Bug Fixes (Do Before Any Production Deploy)
Task 1.1 — Fix re-seeding divergence false flags (admin-matrix-watch.php + all matrix files)
Add a WS_MATRIX_SEEDING_IN_PROGRESS constant guard to ws_matrix_watch_detect_divergence(). In each matrix file (jurisdiction-matrix.php, agency-matrix.php, court-matrix.php, fed-statutes-matrix.php, assist-org-matrix.php), define the constant before any wp_update_post() calls on existing seeded posts. The hook bails immediately if the constant is defined.
Task 1.2 — Fix Accept form stale jx_code / notes (admin-feed-monitor.php)
The Accept form currently passes page-render-snapshot values via hidden fields. Replace them with visible editable text inputs (matching the Save Edits form), or restructure so Accept always reads the currently-saved JSON values and adds a clear UI note that edits must be saved first. The simplest safe fix is replacing the hidden fields with inline text inputs so the user submits what they intend.
Task 1.3 — Decide resolved-flag behavior (admin-matrix-watch.php)
Determine whether ws_matrix_divergence_resolved = 1 means "permanently exempt" or "acknowledged once." If the intent is "acknowledged once," subsequent human edits after resolution should re-flag the post. Update the condition logic accordingly and add a code comment documenting the chosen policy.

Phase 2 — Hardening (Do Before Any Production Deploy)
Task 2.1 — Add LOCK_EX to staged JSON writes (admin-feed-monitor.php)
Change file_put_contents( ws_feed_staged_file(), $json ) to file_put_contents( ws_feed_staged_file(), $json, LOCK_EX ) in ws_feed_monitor_write_staged(). One-line fix.
Task 2.2 — Add index.php guard to feed data directory (admin-feed-monitor.php)
In ws_feed_monitor_ensure_data_dir(), after writing .htaccess, also write an empty index.php (<?php // Silence is golden.) if it doesn't exist. Belt-and-suspenders for Nginx environments.
Task 2.3 — Convert global config map to a function (admin-url-monitor.php)
Replace the file-scope $ws_url_monitor_map global variable with a ws_url_monitor_get_map() function. Remove the global $ws_url_monitor_map declaration from ws_run_url_health_check() and call ws_url_monitor_get_map() instead.
Task 2.4 — Move URL monitor manual trigger to admin_init (admin-url-monitor.php)
Extract the POST handler from ws_url_monitor_render_widget() into a separate function hooked to admin_init. Use a redirect-after-POST pattern (store result in a transient, redirect, then read and display the transient in the widget). Prevents long-running HTTP checks from firing inside the output phase.
Task 2.5 — Add wp_doing_ajax() guard to cron scheduler (admin-feed-monitor.php)
Add if ( wp_doing_ajax() ) return; at the top of ws_feed_monitor_maybe_schedule() to skip the wp_next_scheduled() check on AJAX requests.

Phase 3 — Design Decision
Task 3.1 — Decide ws_ref_type return format (acf-references.php)
Currently return_format => 'label'. If the query layer will ever need to filter references by type, change to 'value' and translate to label at render time. If this field is display-only forever, 'label' is fine. Make the call and add a comment documenting why.

Phase 4 — Housekeeping (Before First Production Commit)
Task 4.1 — Normalize line endings across the plugin
Run dos2unix (or equivalent) on all CRLF files in the plugin. The inconsistency spans most of the older files and both new ACF files. Do this as a single commit with message "Normalize line endings to LF" so it doesn't pollute future diffs.
Task 4.2 — Wire deactivation hooks into ws-core.php
Confirm that ws_feed_monitor_deactivate() and ws_url_monitor_deactivate() are both called from register_deactivation_hook() in the main plugin file. If not, add them.
