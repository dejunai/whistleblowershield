🧱 Phase 0 — Global Architectural Rules

These rules apply to all phases.

🔒 Query Layer Boundary (Non-negotiable)

Shortcodes are presentation only.

They must never call:

get_field()

get_post_meta()

WP_Query

✅ All data must come through:

query-jurisdiction.php

🧠 Shortcode Naming Convention

All shortcode data functions must follow:

ws_get_jx_[entity]_data

Where:

[entity] = child of jurisdiction (e.g. statutes, citations, interpretations)

📌 Enforcement Rules

If a shortcode exists without a matching query-layer function:
→ Create the function

If a shortcode needs additional data:
→ Extend the existing ws_get_*() function

Never bypass the query layer for convenience

⚠️ Relationship System Update

All prior ACF relationship systems have been deleted and scrubbed

Any new relationships must be:

clearly defined

query-layer accessible

not dependent on legacy patterns

🧾 Field Naming Standard

All reference-related fields must use:

ws_reference_*

(No ws_ref_* usage anywhere)

🌐 URL Validation Rule (Global)

Query layer must validate all URLs

If URL is:

missing

malformed

invalid

→ Do NOT return the data object

⚡ Caching Standard (Forward Requirement)

Any new query function must:

follow caching patterns used in query-jurisdictions.php

include a note for standardized caching integration
